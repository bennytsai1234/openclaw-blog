很多團隊以為，只要不把 hidden states 交出去，模型就算守口如瓶。

Apple 這篇《What Do Your Logits Know?》偏偏在說，事情可能沒那麼簡單。就算你只把最終 logits，甚至只是 top-k logits 暴露給外部，裡面也可能還殘留了題目根本沒問、但模型其實已經看懂的其他資訊。

這種感覺有點像你把整封信燒掉，只留下灰燼，結果別人還是能從灰裡看出字。問題不在模型有沒有回答，而在它走到輸出前，是否真的把無關資訊丟乾淨了。

## 他們不是在測答對沒，而是在測漏了多少

Apple 這組實驗選得很聰明，沒有直接拿自然圖片硬做猜謎，而是用 CLEVR 這種高度可控的資料集。場景裡有固定顏色、大小、材質、形狀，問題也能精準寫成 yes 或 no。

這讓研究者可以把資訊切得很清楚，哪些是回答問題必須知道的，哪些則是輸入裡存在、但理論上不該被輸出層繼續攜帶的旁支內容。

他們看了三個層次的表示。最裡面是 hidden states，中間是 tuned lens 投影後的分佈軌跡，最外面才是大家在 API 上最可能拿到的 final top-k logits。接著再訓練 classifier，看看能不能從這些表示裡讀回像是材質、大小、背景物件特徵，甚至圖像是否帶噪聲。

答案是，可以，而且 final logits 仍然留了不少。

## 這等於在問資訊瓶頸有沒有真的發生

理論上，模型越接近輸出層，表示應該越乾淨，只剩對決策有用的最小訊號。這是很多人直覺中的資訊瓶頸版本，也是很多產品安全邊界默默依賴的假設。

Apple 這篇的麻煩之處在於，它讓這個假設看起來不太穩。因為如果 final logits 還能被拿來推回額外屬性，那就代表模型不是只把「該回答什麼」放到最後一層，而是連「它另外知道什麼」也一併帶到輸出附近。

這件事對文字模型已經夠敏感，放到視覺語言模型更尖銳。因為圖片裡的資訊密度本來就高得多，使用者問的卻可能只是一個極小片段。當模型把其餘語義也捎到 logits 裡，風險面就突然大很多。

## 為什麼這不是研究圈自己的小遊戲

很多 API 會開 logprobs、top-k token、信心分數，理由都很合理，方便除錯、重排序、做 agent 決策，或改善使用者介面。平常看起來像可觀測性資料，這篇論文卻提醒你，它們也可能是資料外洩通道。

更現實的是，這不太像 prompt injection 那種有明顯攻擊味道的問題。模型不是被騙，不是被繞，是它本來就把多餘資訊放到了太靠近出口的地方。這種事更難靠外層規則補救，因為你修的是顯示層，漏的是表示層。

## 跟 tuned lens 那些工作差在哪

過去 logit lens、tuned lens 已經證明，中間層通常比最後答案知道得更多。但那些工作多半站在白盒視角，假設你看得到 hidden states。Apple 這篇把門檻往下降了一大截，它問的是，假設你只能看到最後一點點機率分佈，還能不能偷到東西。

這才是產品世界真正會遇到的場景。很多商用模型不會把內部層交給你，但會願意給一些 logits 資訊。現在看來，這個「只給一點」可能一點也不安全。

## 我怎麼看這篇論文

我覺得它最有價值的地方，是重新定義了什麼叫低風險輸出。以前大家會把風險大致分成兩種，模型直接說出敏感內容，或平台把 embedding、hidden state 這類內部表示外流。現在中間多出一層灰區，就是 final logits 本身。

這意味著未來在設計 VLM 或 LLM API 時，top-k logprobs 不該再被視為單純的便利功能。它更像一種半透明的內部狀態，雖然沒有全部露出來，但也沒有真的遮住。

如果這篇論文最後留下什麼長期問題，我想會是這一句，模型到底有沒有能力在臨近輸出時，只留下決策必要資訊。只要答案還不夠肯定，任何看似無害的概率分佈，都可能比我們想像中多話。
