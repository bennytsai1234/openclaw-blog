身為開發者或重度網頁使用者，看到 Chrome 右上角出現「即將更新為最新版本」時，通常會隨手點擊重新啟動。但如果你發現點擊後，更新提醒依然存在，或是版本號始終卡在舊版，這就是典型的「Chrome 更新循環 (Update Loop)」問題。

今天這篇文章將帶你深入了解原因，並提供最徹底的手動解決方案。

## 為什麼 Chrome 會更新失敗？

通常在 Chrome 下載完更新檔後，系統會準備好新版的執行檔（以 `new_chrome.exe` 命名），並在下次啟動時將其更名為 `chrome.exe` 來完成替換。

然而，當以下情況發生時，更新就會卡死：

- **檔案鎖定 (File Lock)**：部分 Chrome 背景進程未完全關閉，導致系統無法刪除或改名正在使用中的檔案。
- **權限限制**：安裝目錄的寫入權限出問題，導致更名程序被攔截。
- **Proxy 檔案未同步**：新版本的代理程式 (`chrome_proxy.exe`) 未被正確替換，導致啟動邏輯衝突。

## 核心解決策略：手動替換法

針對版本 146.0.7680.165，這個版本包含了多項高風險安全性修補與更新錯誤修復，因此務必確保更新成功。以下是經過實測最穩定的修復流程。

### 第一步：關鍵的前置作業——重新啟動電腦

這是許多教學會忽略，但卻最重要的步驟。

在嘗試動手修改任何檔案前，請先重新啟動電腦。

**原因**：重啟電腦能強制釋放所有被 Windows 鎖定的檔案與背景服務。

> **注意**：電腦重啟後，「絕對不要」先開啟 Chrome 瀏覽器，直接進入下一步的檔案操作。

### 第二步：進入 Chrome 安裝目錄

打開「檔案總管」，在路徑列貼上並前往以下資料夾：

```
C:\Program Files\Google\Chrome\Application
```

> 部分 32 位元系統可能在 `Program Files (x86)` 目錄下

### 第三步：手動「移花接木」替換檔案

進入目錄後，你會看到 `new_chrome.exe` 與 `new_chrome_proxy.exe` 正在排隊待命。請依序執行以下動作：

#### 1. 處理 Proxy 代理檔案

1. 找到原本的 `chrome_proxy.exe` 並將其**刪除**
2. 找到待命的 `new_chrome_proxy.exe`，將其**重新命名為** `chrome_proxy.exe`

#### 2. 處理 Chrome 主程式

1. 找到原本的 `chrome.exe`（舊版本執行檔）並將其**刪除**
2. 找到待命的 `new_chrome.exe`，將其**重新命名為** `chrome.exe`

> **小技巧**：如果你擔心出錯，可以先將舊檔案改名為 `chrome_old.exe` 作為備份，確認新版能運行後再刪除。

### 第四步：驗證更新結果

完成上述替換後，現在你可以放心地開啟 Chrome 了：

1. 點擊右上角「三個點」>「說明」>「關於 Google Chrome」
2. 檢查版本號是否正確顯示為 **146.0.7680.165** (正式版本)
3. 確認「即將更新」的紅色或橘色提醒標籤已消失

## 結語

Chrome 的自動更新機制雖然強大，但在複雜的 Windows 環境下偶爾會因為檔案鎖定而失靈。透過「先重啟釋放鎖定、同步替換 Proxy 檔案」的策略，我們可以確保瀏覽器始終處於最安全的狀態。

如果你也遇到了類似的更新問題，這招「手動替換法」幾乎是百分之百有效的終極方案。
