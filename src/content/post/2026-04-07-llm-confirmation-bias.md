## 這篇文章在說什麼

想像你有三個數字：2、4、6。我知道一個秘密規則符合這組數字，你的任務是透過不斷猜測並測試新的三數組合，來發現這個規則。每次你提出一組數字，我只告訴你「符合」或「不符合」。這是心理學家 Wason 在 1960 年設計的經典實驗，用來測量人類的確認偏誤——人們傾向於提出「符合自己假說」的組合來測試，而不是去「推翻」自己的假說。

NYU 的研究團隊想知道：LLM 會不會也這樣？

他們把這個實驗改成了可以讓 AI agent 互動的形式，11 個不同規模、不同家族的 LLM 都被拿來測試。結果發現，所有模型都展現出明顯的確認偏誤——它們提出新測試案例時，選擇「支持自己當前假說」的比例遠高於「嘗試推翻」的，導致規則發現效率低落。

但這篇論文最有趣的地方是：他們沒有重新發明一套干預方法，而是直接借用了給人類使用的兩種心理學策略——**Think-in-Opposites**（反向思考）和 **Dual-Goal**（雙目標）——結果讓 LLM 的規則發現率從 42% 提升到 56%。之後再透過知識蒸餾把這些策略蒸餾進模型，模型還能泛化到全新的領域（Blicket Test），不需要每次推論時都加上 prompt。

## 為什麼重要

確認偏誤聽起來像心理學課題，跟工程師有什麼關係？

關係大了。當 LLM 被當作 agent 使用——规划任务、调用工具、多步推理——它們並不是靜態回答問題，而是要不斷產生假設、測試、修訂。如果模型天生就傾向「只找支持自己的證據」，那它在複雜推理鏈中犯錯的機率會高得多，而且錯誤會一路累積，最後輸出一個自信但完全錯誤的答案。

這篇論文用實證方式量化了這個問題的嚴重程度：不同模型的任務成功率從 6% 到 78% 不等，而且確認偏誤越嚴重的模型，任務成功率越低。這告訴我們，光靠 Scale UP 並不能自動解決這個問題——需要專門的設計。

更值得關注的是，干預手段的有效性暗示了另一條路：與其訓練更大的模型，不如訓練「更會懷疑自己」的模型。

## 技術細節

**實驗設計**：研究團隊採用 Wason 2-4-6 任務的變體。每個 episode 有一個初始三元組和一個隱藏規則，LLM 在 45 個互動回合內透過不斷猜測和測試來發現規則。每回合分為「猜測」（Guess）和「測試」（Test）兩個階段，後者提出新的三元組，環境返回二元 feedback（是否符合規則）。

**確認偏誤量化**：用 I:C（Incompatible:Compatible Ratio）指標——模型提出「推翻假說」的測試次數，除以「支持假說」的測試次數。數值越高，代表模型越願意嘗試推翻自己的假說，也就是偏誤越低。

**干預策略**：
- **Think-in-Opposites**：指示模型「找出一個顯著特徵，並故意測試與該特徵相反的實例」
- **Dual-Goal**：告訴模型要同時猜測 DAX 規則（原始規則）和 MED 規則（DAX 的補集），迫使它同時考慮支持和反駁的證據

**知識蒸餾**：用符號蒸餾（symbolic knowledge distillation）把干預策略的行為蒸餾進 LLM，使其在沒有額外 prompt 的情況下也能表現出反確認偏誤的推理模式。蒸餾後的模型在全新的 Blicket Test 任務上表現出正向遷移，確認了泛化能力。

## 我的觀點

這篇論文最讓我印象深刻的是「借用人類工具」這個思路。通常當我們討論 LLM 的認知缺陷時，直覺會去找 NLP 或機器學習領域的解決方案——新的訓練目標、新的架構、更多的數據。但這群人跑去翻了心理學文獻，把幾十年前就設計好的人類干預策略拿來用，效果還不錯。

這暴露了一個我們經常忽略的事實：LLM 的很多「推理問題」並不是數學上或計算上的瓶頸，而是認知層面的——人類歷史上踩過的坑，LLM 也會踩同樣的坑。確認偏誤不是 LLM 特有的缺陷，而是智慧系統在有限資訊下進行推理時的自然傾向。

但我也有保留意見：2-4-6 任務是一個高度結構化的實驗室任務，現實中的推理問題往往沒有這麼乾淨的反饋迴路。在真實應用場景裡，「嘗試推翻自己」的成本可能比實驗室高得多——你問醫生「你確定這是對的診斷嗎？」和模型多跑一個測試案例，代價是完全不同的。

這篇論文開了一扇門，但門後的世界還需要更多探索。
