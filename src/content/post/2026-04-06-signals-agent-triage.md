凌晨三點，監控儀表板上的數字還在瘋狂跳動。你新上線的 AI Agent，在過去 24 小時內，已經處理了超過一百萬次與用戶的互動。伺服器日誌像瀑布一樣滾動，每一行都是一個 JSON 物件，記錄著一次完整的對話軌跡。你的老闆明天早上九點想知道：「它表現得怎麼樣？」你該如何回答？

你不可能讀完一百萬條日誌。隨機抽樣幾條來看？那就像在太平洋裡撈一根特定的針。這就是今天所有 Agent 開發團隊面臨的困境：當系統部署後，我們被自己產生的數據洪流所淹沒，卻不知道哪一滴水才最重要。

過去，我們只有兩種笨方法。第一種是人海戰術：標註團隊逐條閱讀評分。這方法雖準，但極其昂貴且緩慢，遠跟不上迭代速度。第二種是用更強的 LLM 當裁判，自動化評估。這方法成本依然不低，且引入了新問題：誰來評判裁判？當裁判自身都不可靠時，我們只是在昂貴、緩慢且不確定地打撈那些「出錯」的軌跡。Post-deployment 最佳化，這個看似光鮮的詞，實則成了開發流程中的巨大瓶頸。

Katanemo Labs 的一篇新論文《Signals》（arXiv:2604.00356）沒有提出一個更聰明的裁判，而是釜底抽薪地問了一個完全不同的問題：我們真的需要一個「裁判」嗎？會不會，軌跡本身就已經用最樸素的方式，告訴了我們它是否「有趣」？這篇研究的核心洞見是：與其花大錢請 LLM 來做主觀判斷，不如先從互動記錄中提取那些不需要任何智能就能計算的、客觀的「信號」。

### 軌跡自己會說話，如果你懂得傾聽

這些「信號」是什麼？它們不是什麼高深的魔法，而是一些我們在除錯時憑直覺就會關注的特徵。研究者將它們歸納成幾個大類：

首先是**互動層面的信號**。有時候，Agent 明顯誤解了用戶的意圖，答非所問（**Misalignment**）；有時候，它像鬼打牆一樣，在同一個問題上反覆嘗試同樣的無效操作，無法取得進展（**Stagnation**）。更糟的是，用戶或 Agent 可能直接放棄了對話，揮手告別（**Disengagement**）。當然，也有最好的情況：任務圓滿成功（**Satisfaction**）。

其次是**執行層面的信號**。這些更加具體，像是 Agent 嘗試呼叫一個工具，結果 API 回傳了錯誤（**Failure**），或者它陷入了一個無盡的重複動作循環中（**Loop**），不斷執行著同樣的幾個步驟。

最後是**環境層面的信號**。這就像汽車沒油了，Agent 可能因為達到了 Token 上限、API 請求次數耗盡等資源限制而被迫中止（**Exhaustion**）。

這些信號有一個共同的、也是最關鍵的特點：**計算它們的成本趨近於零，完全不需要再次呼叫任何 LLM**。它們是軌跡數據中與生俱來的結構化特徵，是客觀存在的事實，而非某個裁判的主觀意見。這套分類方法被設計為「廣泛適用」的，無論你用的是哪個模型、在哪個領域，這些基礎的互動模式都普遍存在。

### 所以，靠信號篩選真的比較有效嗎？

空談無益，數據最有說服力。研究團隊在一個專門評估工具增強型 Agent 的標準測試集 τ-bench 上進行了實驗。結果相當驚人：

如果完全隨機抽樣，你找到一條「有信息量」（能幫助你改進系統）的軌跡的機率是 **54%**——比拋硬幣好不了多少。
如果用一些傳統的、基於規則的啟發式方法（例如，只看那些包含「錯誤」字眼的日誌），這個比率能提升到 **74%**。
但如果使用 Signals 框架來指導抽樣，信息率直接飆升到 **82%**。

這意味著，透過聆聽軌跡自身的信號，我們識別高價值樣本的準確率遠高於盲目的大海撈針。更重要的是效率的提升。研究計算出，每找到一條高價值軌跡，Signals 方法的效率是隨機抽樣的 **1.52 倍**。這不僅僅是省錢，更是省下了開發團隊最寶貴的資產：時間和專注力。團隊不再需要浪費精力在那些平淡無奇、無法提供新知的互動上，而是可以集中火力去分析那些真正能揭示系統缺陷或亮點的關鍵案例。

### 信號是答案，但不是所有問題的答案

《Signals》這篇研究最讓我欣賞的，是它對問題的重新定義。它巧妙地避開了「如何更準確地評判」這個無底洞，轉而探索「哪些信息是客觀且廉價的」。在整個行業都在為 LLM 裁判和人類評分的不可靠性而頭痛時（一個有趣的對照是，Google 的研究也曾指出人類評分者之間的一致性問題），這種回歸基本、從結構中找答案的思路，顯得格外清醒和實用。停滯、循環、資源耗盡——這些是工程師一眼就能看明白的事實，不需要另一個 AI 來指手畫腳。

然而，我們也須看清其邊界。首先，實驗僅限於 τ-bench 這類結構化任務。在創意寫作、長篇推理等更主觀的場景，這些信號是否依然有效，仍是未知數。「停滯」在工具任務中是壞事，在腦力激盪中卻可能是過程。其次，「信息量」的定義因團隊而異。追求穩定的團隊關注失敗，探索新功能的團隊可能更在意用戶為何放棄。Signals 提供了強大的篩選器，但「什麼值得看」的最終判斷，仍取決於人。

最終，《Signals》就像在喧囂的數據工廠裡，為我們安裝了一套高效的分類傳送帶。它不能代替工程師的分析和判斷，但它能確保送到你面前的，都是最值得你花時間檢查的零件。它提醒我們，在追逐更大、更強模型的同時，或許也該回過頭來，學習如何更聰明地理解我們已經擁有的東西。畢竟，在百萬次迷航之後，找到那幾條能指引方向的航海日誌，可能比造一艘更快的船來得更重要。
