## 1. 專案總覽

Gemini CLI 是 Google 開源的 AI Agent 工具，直接把 Gemini 模型帶進終端機。採用 ReAct（Reasoning + Acting）循環，結合本地工具與 MCP（Model Context Protocol）伺服器，支援除錯、寫程式、自動化腳本與深度研究等多種場景。最新版本 `0.37.0` 仍為 Preview 階段，穩定版為 `0.36.0`。

目前版本分布（截至 2026-04-09）：

| 版本類型 | 最新版本 | 備註 |
|---------|---------|------|
| Stable | 0.36.0 | 238 次下載/天 |
| Preview | 0.37.0-preview.2 | 2026-04-07 發布 |
| Nightly | 0.39.0-nightly.* | 2026-04-09 最新 |

**安裝方式：**

```bash
# 全域安裝
npm install -g @google/gemini-cli

# 或直接用 npx 運行
npx @google/gemini-cli

# 用 mise 管理版本
mise use gemini-cli@latest
```

## 2. 核心功能解析

### 2.1 上下文感知持久化政策審批（⭐ 最重要）

```typescript
// PR #23257 - feat(core): context-aware persistent policy approvals
// 政策審批現在會根據目錄上下文自動記住，不再每次都問
```

這是 0.37.0 最重要的功能。之前的版本每次開新專案，危險操作（如刪除檔案、執行 Shell 命令）都要重新確認。現在系統會根據專案目錄上下文自動記住審批結果，大幅減少重複授權的煩躁感。

**實測情境：**
```
場景 1：openclaw-blog/ 下執行 rm -rf ./node_modules（已審批一次）→ 直接執行
場景 2：陌生專案目錄下執行同樣指令 → 仍需確認
```

底層原理是根據專案路徑建立信任等級映射，危險操作在信任目錄內自動放行。

### 2.2 TerminalBuffer 模式（解決閃爍）

```bash
# PR #24512 - feat(ui): enable "TerminalBuffer" mode to solve flicker
# 滑動時不再閃爍，UI 更順暢
```

```json
// settings.json
{
  "ui": {
    "terminalBufferMode": true
  }
}
```

過去在終端機滑動時畫面會閃爍，特別是輸出大量內容時。0.37.0 引入 TerminalBuffer 渲染模式，解決這個長期困擾。

### 2.3 專案記憶體目錄移至 tmp

```
// PR #24542 - fix(core): move project memory dir under tmp directory
// 隱私強化：專案特定資料不再殘留在主目錄
```

`.gemini/` 資料夾從專案根目錄移至系統 tmp 目錄，避免敏感資訊意外進 Git。

### 2.4 工具輸出截斷（實驗性）

```json
// PR by @SandyTao520
{
  "enableToolOutputTruncation": true
}
```

當 Shell 工具輸出過長時，自動截斷並儲存完整結果到檔案，適合自動化腳本場景。

## 3. 實際應用案例

### 3.1 日常開發自動化

```bash
# 非互動模式 - 適合 CI/CD
gemini -p "Explain this error: $(cat error.log)"

# 或 pipe 方式
echo "What is fine tuning?" | gemini
```

### 3.2 MCP 整合

```bash
# 連接遠端 MCP 伺服器
gemini /mcp connect https://mcp-server.example.com

# 初始化時顯示 MCP 載入進度條
# PR by @swissspidy - MCP loading indicator
```

### 3.3 會話統計分析

```bash
# 輸出詳細 Session 統計到 JSON 檔案
gemini --session-summary ./session-stats.json

# 可用於分析 API 消耗、最佳化 Prompt
```

## 6. 優缺點分析

### 優點

- **✅ Context Window 大**：1M tokens，領先對手
- **✅ 完全開源**：Apache 2.0，可自架、可改寫
- **✅ MCP 原生支援**：生態系擴展性強
- **✅ 政策審批持久化**：大幅減少重複確認
- **✅ Cloud Shell 內建**：GCP 用戶零設定開箱即用

### 缺點

- **⚠️ Preview 版本穩定性**：0.37.0 仍為預覽，生產環境建議用 0.36.0
- **⚠️ Chocolatey 測試失敗**：社群回報 v0.37.0 在 Chocolatey 上的套件測試有失敗記錄
- **⚠️ Windows 仍有小問題**：vim 路徑找不到時的崩潰問題（#22423）才剛修復

## 7. 常見問題 FAQ

**Q：0.37.0 跟 0.36.0 差在哪？**
A：0.37.0 是 Preview 版本，主要新功能是政策審批持久化與 TerminalBuffer。如果追求穩定，用 0.36.0；如果想體驗新功能，可以用 0.37.0-preview.2。

**Q：Chocolatey 安裝失敗怎麼辦？**
A：改用 npm 或 npx 安裝，或等下一個 patch 版本。目前社群已回報這個問題。

**Q：可以只更新不重新登入嗎？**
A：`npm install -g @google/gemini-cli` 更新後無需重新登入，OAuth token 會保留。

**Q：如何關閉政策審批持久化？**
A：目前沒有官方開關，但可以透過清理 `~/.gemini/memory/` 或 `settings.json` 中的信任目錄設定來達到類似效果。

## 9. 安全與風險

### 9.1 信任資料夾設定

```
# settings.json - Trusted Folders
{
  "trustedFolders": [
    "/home/benny/projects",
    "~/openclaw-blog"
  ]
}
```

### 9.2 政策審批的安全影響

政策審批持久化是雙面刃——便利性提升，但風險也在信任目錄內累積。建議：

1. 不要把高風險專案（如金融、醫療）加入信任清單
2. 定期檢視 `~/.gemini/memory/` 資料
3. 生產目錄建議維持互動式審批模式

## 11. 個人觀點

Gemini CLI 0.37.0 最讓我驚喜的不是單一功能，而是整個產品的方向感——從 0.28 到 0.37，553 個版本，每一天幾乎都有新東西。

**三個值得關注的趨勢：**

1. **CLI 正在重新定義開發者工作流**：傳統 CLI 是工具，今天的 Gemini CLI 是 Agent。差異在於工具是被動的，Agent 是主動規劃並執行的。

2. **持久化政策審批是殺手級功能**：這個功能看起來很小，但實際體驗差異巨大。每次 `rm -rf` 都要按一次確認，累積起來是巨大的摩擦。

3. **開源 + 大 Context Window 是核心差異化**：相較於 Claude CLI 和 OpenAI CLI，Gemini CLI 的開源協議與 1M tokens 上下文窗口，在特定場景（大型程式碼庫分析、深度研究）有不可替代的優勢。

我個人的使用習慣：日常除錯用 Claude CLI（反應快），大型重構用 Gemini CLI（1M tokens 吃到飽），自動化腳本用 Gemini CLI Headless Mode（`gemini -p` 真的太方便了）。

## 12. 入門指南

### Step 1：安裝

```bash
# 方式一：npm（推薦）
npm install -g @google/gemini-cli

# 方式二：npx（免安裝）
npx @google/gemini-cli

# 方式三：mise
mise use gemini-cli@latest
```

### Step 2：設定 API Key

```bash
# 方式一：直接設定
export GEMINI_API_KEY="your-api-key-here"
gemini

# 方式二：Google 登入（推薦，有免費額度）
gemini
# 選擇 "Sign in with Google"
```

### Step 3：基礎操作

```bash
# 進入互動模式
gemini

# 單次查詢（非互動模式）
gemini -p "Explain what is Model Context Protocol"

# 查看幫助
gemini /help

# 查看統計
gemini /stats

# 查看可用工具
gemini /tools
```

### Step 4：設定 Trusted Folders

```bash
# 建立設定檔
mkdir -p ~/.gemini
cat > ~/.gemini/settings.json << 'EOF'
{
  "trustedFolders": [
    "~/projects",
    "~/openclaw-blog"
  ],
  "ui": {
    "terminalBufferMode": true
  }
}
EOF
```

### Step 5：探索 Agent Skills

```bash
# 進入 Agent Skills
gemini /skills

# 使用內建 Skills（v0.37.0 新增內建 agent skills）
```
