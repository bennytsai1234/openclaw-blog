## 這篇文章在說什麼

2026 年 4 月，Meta AI 與沙烏地阿拉伯阿卜杜拉科技大學（KAUST）發表了一篇論文，提出一種全新的機器形式：**Neural Computers（NCs）**。核心理念很直接——與其讓 AI Agent 操作外部軟體堆棧（作業系統、終端機、API）來完成任務，不如讓神經網路本身就變成一台可運作的電腦。

這不是在模型外面包一層殼，而是把「運算」這件事整個內化：執行環境、工作記憶體、介面狀態，這些本來由作業系統管理的東西，全部寫進模型的潛在狀態（latent state）裡。論文明確與過往技術拉開距離——不是 Neural Turing Machine 的外部記憶體架構，不是 World Model 的環境預測，也不是 Agent 的工具呼叫，而是**模型即電腦**。

最終目標稱為 Completely Neural Computer（CNC），必須同時滿足四個條件：圖靈完整、普遍可程式化、行為一致性（除非明確重新程式化，否則表現一致）、以及機器原生的架構與程式語言語義。

## 為什麼重要

這條路徑試圖回答一個根本問題：AI Agent 真的有「執行」能力嗎？

現在的 Agent 本質上是一個調用者——它接受指令，然後透過 API、終端機、瀏覽器操作外部軟體。模型本身不執行任何東西，只是決策層。這在淺層任務上很有效，但一旦需要長時、可組合、具狀態的執行，就會遇到瓶頸：外部工具有狀態漂移問題，API 相容性問題，還有跨環境的一致性問題。

NCs 嘗試繞過這層障礙，把執行責任直接吃進模型。如果成功，硬體裝置的運算邊界就不再由作業系統決定，而由模型的能力決定。這代表一種完全不同的計算範式——不再有作業系統、沒有容器、沒有 POSIX 相容性問題，理論上只要一個足夠大的模型就可以運作任意軟體。

當然，論文本身也承認，離這個願景還非常遠。

## 技術細節

### 兩個原型

研究團隊在 **Wan2.1**（發表當時的 SOTA 影片生成模型）之上建構了兩個原型：

**NCCLIGen**：處理命令列介面（CLI）互動。模型接收文字標題與第一個畫面格，輸出後續終端機幀。影片生成被當作「文字＋圖片 → 影片」來處理：CLIP 編碼器處理第一個畫面格，T5 編碼器處理標題，條件特徵與擴散噪聲拼接後送入 DiT（Diffusion Transformer）堆疊。

訓練資料有兩組：CLIGen（General）約 823,989 條影片流（約 1,100 小時），來自公開的 asciinema.cast 錄製；CLIGen（Clean）約 78,000 條常規軌跡加上約 50,000 條 Python 數學驗證軌跡，用 vhs 工具在 Docker 環境裡自動生成。

訓練成本：CLIGen（General）約 15,000 張 H100 GPU 小時；CLIGen（Clean）約 7,000 張 H100 GPU 小時。

**NCGUIWorld**：處理完整桌面圖形介面。每一筆互動都是 RGB 幀與輸入事件的同步序列，解析度 1024×768，在 Ubuntu 22.04（XFCE4）上以 15 FPS 收集。資料集總計約 1,510 小時：Random Slow（~1,000 小時）、Random Fast（~400 小時），以及 110 小時由 Claude CUA 收集的目標導向軌跡。訓練使用 64 張 GPU 約 15 天，單次完整跑約 23,000 GPU 小時。

### 關鍵數據

NCCLIGen 在 CLIGen（General）上的重建品質：平均 PSNR 40.77 dB，SSIM 0.989（字體大小 13px）。字元級準確度從初始的 0.03 提升到 60,000 步時的 0.54，完整行比對準確度達 0.31。

一個值得注意的發現：PSNR 與 SSIM 在 25,000 步左右就趨於 plateau，即使訓練到 460,000 步也沒有實質提升。這與許多視訊生成模型的 Scaling 行為不太一樣。

在符號計算方面，團隊測試了 1,000 道數學題的準確度：NCCLIGen 4%，基礎 Wan2.1 0%，作為對照的 Sora-2 71%，Veo3.1 2%。有趣的是，如果直接在 prompt 裡提供正確答案，NCCLIGen 的準確度從 4% 跳到 83%——代表模型是**可引導的**，而非真的內建了算術能力。

NCGUIWorld 測試了四種動作嵌入方式：external、contextual、residual、internal。Internal conditioning（在每個 Transformer 區塊內插入動作交叉注意力）達到最佳結構一致性（SSIM+15 = 0.863，FVD+15 = 14.5）。殘差條件效果最佳的是知覺距離（LPIPS+15 = 0.138）。

滑鼠控制實驗：SVG 遮罩 / 蔘照條件將游標準確度拉到 98.7%，遠優於純座標監督的 8.7%——說明把游標當成一種明確的視覺物件來監督是關鍵。

最後一個數據很重要的結論：110 小時的 Claude CUA 精心策劃資料，在所有指標上都打敗了約 1,400 小時的隨機探索（FVD：14.72 vs. 20.37 vs. 48.17）。高品質、目标导向的資料遠比被動收集更有效率。

## 我的觀點

這篇論文最讓人影響深刻的地方不是原型展示的東西，而是問題意識。

AI Agent 的極限在 2025–2026 年已經被大量討論過——工具呼叫的可靠度、狀態管理的複雜度、長時任務的一致性。研究者選擇從根源重新思考「什麼是執行」這個問題，而不是繼續在 Agent 框架上最佳化，這需要勇氣。

但我認為距離實際應用還很遠。兩件事特別關鍵：

**第一，符號計算能力仍然是封閉的。** 模型在數學推理上靠 prompt 提示才能達到 83% 準確度，這與「圖靈完整、普遍可程式化」的目標有巨大落差。沒有辦法穩定執行符號運算，就不可能當作通用電腦。

**第二，行為一致性與長期執行穩定性完全沒有被解決。** 25,000 步就 plateau 的觀察讓人不安——如果模型學到的 routine 無法長期穩定重用，應用場景就會非常受限。

不過，作為一個研究方向，NCs 的價值在於它建立了一個新的設計空間——過去所有對「模型作為决策者」的討論，現在有了一個「模型作為執行者」的對立面。這個問題本身值得持續關注。
