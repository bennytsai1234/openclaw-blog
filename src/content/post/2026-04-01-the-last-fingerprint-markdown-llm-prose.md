有些 AI 文章，你只看兩段就知道不對勁。不是它寫得差，甚至常常還算流暢，但那種句子中間忽然插進來的破折號，像一種揮之不去的口音。久了之後，大家開始拿它當笑話，也拿它當檢測器。可是真正有趣的問題不是「它又用了破折號」，而是它為什麼這麼難戒。

The Last Fingerprint 這篇論文做的，就是把這個小小的標點一路往回追。作者不是停在風格評論，而是去問，這到底是訓練資料的結果，還是後訓練的偏好，還是模型在學寫作時留下的一種結構性指紋。這個題目看似小，讀完卻會讓你重新理解 LLM 的輸出行為。

## 破折號真的是壞習慣嗎

如果把問題只看成「模型愛裝文青」，那就太淺了。論文的核心觀點是，破折號之所以頻繁出現，不只是因為模型模仿了某些作者，而是它長期浸泡在 Markdown 主導的訓練資料裡。GitHub、Reddit、Stack Overflow、各種文件站，都是高品質文字來源，也都高度結構化。這些資料不停教模型一件事，dash 經常代表邊界、轉折、列表、段落分隔。

久而久之，模型內化的不是單一符號，而是「橫線意味著結構」。所以就算你把標題、粗體、條列清掉，那種想替句子加出一個結構轉點的衝動，還是會從 prose 裡冒出來，而 em dash 剛好是最像正常標點、又還保有結構感的選擇。

## 為什麼禁止 Markdown 還是沒用

論文做了一個很漂亮的對照。作者讓 12 個模型分別在自由生成和「不要用 Markdown」兩種條件下寫作，然後去算每千字的破折號數量。結果是，標題、列表、粗體這些外顯格式的確幾乎都消失了，但破折號沒有一起消失。

GPT-4.1 在被要求不用 Markdown 後，仍然每千字出現 9.1 個破折號。DeepSeek 也還很高。相反地，Llama 幾乎是零。這組數據有兩層意思。第一，破折號不是單純的格式殘留，而是更深的寫作習慣。第二，不同模型供應商在後訓練上做了不同選擇，有人強化這種結構感，有人把它壓掉了。

這也是論文標題裡「最後的指紋」真正想說的事。當最明顯的 Markdown 痕跡都被洗掉後，破折號成了最後一個還留在表面的東西。

## 問題不只在預訓練，還在後訓練

我覺得最值得看的，是作者把 post-training 拉進來解釋。很多人以為 RLHF 只是讓模型更有禮貌、更安全，但它其實也會塑造文體。評估者通常偏好看起來有條理、節奏分明、專業感夠的文字，而破折號正好能快速製造這種閱讀幻覺。它讓句子像是先想好了架構，再往裡面填內容。

於是模型學到的不是「破折號很美」，而是「這種寫法比較容易被判成好回答」。這就是為什麼有些 instruct 模型反而比 base 模型更容易出現這種痕跡。不是它們比較不自然，而是它們更努力迎合某種被偏好的清晰感。

## 這能拿來做 AI 內容辨識嗎

某種程度上可以，但我不會把它當萬用武器。論文給出的啟發是，輸出風格可以被當成模型訓練方法的側寫。你不一定能單靠破折號判斷一段文字是不是 AI 寫的，因為人類作者也會用，可是當你觀察的是整體頻率、在不同提示下的殘留情況、以及和其他格式特徵一起出現的模式時，它就變得有辨識力。

更重要的是，這種方法比討論「AI 有沒有靈魂」實際得多。你盯住一個可測量的小行為，然後去追它背後的資料結構與獎勵機制，模型就不再是黑盒。

## 我更在意的是另一件事

讀這篇論文時，我一直想到一個更大的問題。我們常說模型要寫得自然，卻很少追問，誰定義自然。現在很多模型看起來像研究員、顧問、工程師混合體，句子有結構、有控制感，也因此帶著一種明顯的機器腔。破折號只是其中一個洩漏點。

真正被暴露的，其實是後訓練對「好文章」的審美。只要獎勵機制繼續偏好整齊、清楚、可掃描、像文件一樣的文字，那麼就算禁掉破折號，也還會有別的結構癖跑出來。

所以這篇論文真正迷人的地方，不是它揭穿了破折號，而是它示範了一種研究方式。從一個微小的輸出習慣下手，一路追到資料來源、結構內化、再到 RLHF 偏好。這條路很慢，但比只看排行榜分數更能理解模型到底變成了什麼樣子。
